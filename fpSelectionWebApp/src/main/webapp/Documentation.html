FP Selection

The Fluorescent Protein Selection tool (shown below) takes in the user's cytometer settings and a full list of possible fluorophores, including their excitation and emission spectrums, and optimally selects a setup for the user based on how many fluorophores they would like to use, indicated by the input 'N'. In the result generated, it matches N chosen fluorophores with N individual filters for the user to use. If N exceeds the number of filters, it selects a fluorophore for all of the filters. If N exceeds the number of fluorophores, it will use as many of the fluorophores as possible.
  • Exhaustive (Slow) - This option takes the longest amount of time to generate a result for larger 'N', but provides the best result
  • Beam Width Search (Medium) - This option utilizes preprocessing of fluorophores to speed up the search time. Provides similar if not the same result as Exhaustive. The size of the list of fluorophores to search is determined by the percentage input by the user
  • Hill Climbing (Fast) - This option preprocesses the list of fluorophores and trims the full list down to 'N' to provide a result to the user. Provides similar if not the same result as the above two, but for large N the selection is least optimal. However, for large lists of fluorophores and large N, this option provides the fastest response to the user.

Laser Selection

The Laser Selection tool (shown below) takes in the user's filter settings (in the form of a formatted cytometer csv file) and a full list of possible fluorophores, including their excitation and emission spectrums, and based on the input 'N' fits up to N lasers with the user's given information. In the result generated, the algorithm provides a sample setup with the generated number of lasers. The algorithm divides the visible spectrum into N sections, and creates a sample laser in each section. Varying their wavelengths to scan a section of the spectrum, a laser is only considered if there is considerable response from a given fluorophore in that laser. If less than N lasers are generated, the algorithm achieved better results with each filter and fluorophore by matching them with similar laser wavelengths, and received lower Signal to Noise ratios by using N lasers.

Filter Selection

The Filter Selection tool (shown below), takes in the user's laser settings (in the form of a formatted cytometer csv file) and a full list of possible fluorophores, including their excitation and emission spectrums, and based on the input 'N' fits up to N filters with the user's given information. The algorithm systematically fits varying width filters with each fluorophore if the fluorophore is responsive above a certain threshold in a given laser. In the result generated, the algorithm provides a setup with the generated number of filters. If less than N filters are generated, the algorithm achieved worse results by using more than the shown number of filters.

"My" FP

The "My" FP tool (shown below), takes in the user's cytometer settings and a list of all fluorophores being used, including their excitation and emission spectrums. Using these files, the selection algorithm will run and produce what it believes to be the optimal setup for that combination of settings and fluorophores. The algorithm assumes that the number of fluorophores to select, 'N', is equal to the total number of fluorophores present.
